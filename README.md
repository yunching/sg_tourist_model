# Singapore tourist pressure MVP

This project forecasts monthly visitor arrivals from Singapore's ten largest or strategically important source markets, then converts those forecasts into a simple tourism-pressure index.

Markets included:

- China, Indonesia, Malaysia, Australia and India
- Japan, South Korea, Philippines, United Kingdom and United States

The model uses Singapore Tourism Board arrival data published through data.gov.sg, national public-holiday calendars, seasonality and lagged arrivals. School holidays and customary travel periods can be added through a versioned manual input file.

## What the MVP produces

Running the pipeline creates:

- `data/raw/arrivals.csv`: normalized monthly arrivals by market
- `data/raw/visitor_days.csv`: Singapore's total monthly visitor days
- `data/processed/holiday_features.csv`: monthly holiday features by origin
- `data/processed/model_panel.csv`: training panel
- `outputs/evaluation.json`: rolling holdout metrics against a seasonal-naive baseline
- `outputs/market_forecast.csv`: 12-month forecast by market
- `outputs/tourism_pressure_forecast.csv`: estimated total arrivals, average tourists present and a pressure index where the 2019 monthly mean is 100

The pressure output is a tourism-pressure indicator, not a literal pedestrian count.

## Setup

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -e .
```

On macOS or Linux, activate with `source .venv/bin/activate`.

## Run

```powershell
sg-tourist-model run
```

Useful options:

```powershell
sg-tourist-model run --forecast-months 12 --holdout-months 12
sg-tourist-model run --no-refresh
```

`--no-refresh` reuses cached raw CSV files. A normal run refreshes official data from data.gov.sg.

## Add school holidays and travel periods

Edit `data/manual/travel_windows.csv`. Each row is a dated window for one modeled market:

```csv
market,start_date,end_date,name,kind,weight,source_url
Japan,2026-08-13,2026-08-16,Obon,customary_break,1.0,https://example.gov/source
```

Use exact market names from the file header comments and retain the official source URL. The pipeline expands each window to daily observations and creates monthly weighted break-day features. An empty file is valid.

## Modeling choices

- The target is the bounded log-change from the same month one year earlier. This retains a strong seasonal anchor while allowing holiday, trend and recent-arrival features to adjust it.
- Public holidays are converted into travel-opportunity blocks by joining them to the locally observed weekend. The model counts effective days off, long weekends, maximum block length, departure/return spillover across month boundaries and overlap with school breaks.
- Johor uses its historical Friday-Saturday weekend through 2024 and Saturday-Sunday from 2025. Johor-only holidays remain separate from Malaysia-wide holidays.
- China weekend workdays used to compensate for longer official breaks are removed from time-off blocks when the source calendar identifies them. Manual working_day rows can supply other official overrides.
- A pooled ridge regression shares a general holiday response while regularized market interactions allow each origin to react differently.
- Holdout WAPE selects the production forecast automatically; the seasonal-naive model remains the champion whenever the holiday-adjusted challenger fails to improve on it.
- 2020–2023 are excluded from fitting by default because border controls and the reopening ramp overwhelm normal calendar effects.
- Ridge shrinkage is selected on a trailing validation slice inside the training period; the reported holdout remains untouched.
- Evaluation uses the latest complete holdout period and compares the model with a 12-month seasonal-naive forecast.
- Twelve-month forecasts are recursive, so later forecast months carry more uncertainty.

## Data sources

- Arrivals by place of residence: data.gov.sg dataset `d_7e7b2ee60c6ffc962f80fef129cf306e`
- Visitor days: data.gov.sg dataset `d_f1fa8e1ffdda0912360f820af84c2b9e`
- Holiday dates: generated by the `holidays` package; country-specific official calendars should be archived when adding manual corrections or school breaks

## Known limitations

- Official arrival and visitor-day targets are monthly, so this does not validate day-level crowding.
- National calendars are imperfect proxies for Australia, India, Malaysia, the UK and the US, where holidays or school breaks vary regionally.
- China's compensating workdays, Indonesia's collective leave and one-off holiday changes should be supplied through the manual input when they are not represented by the holiday library.
- Events, air capacity, prices, weather and hotel forward bookings are not yet included.
